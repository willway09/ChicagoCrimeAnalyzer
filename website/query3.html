Query 3: Crimes By Type

What is the total number of crimes of a given IUCR or NIBRS code in the nth month of the year over the given year interval? Do any trends emerge?

IUCR and NIBRS are codes which classify crimes at either the state or federal level. In either case, this query would select only crimes of the given code, and output the total times they occur in a given month of the year (for example just in May) as a function of the past n years. Suppose n = 5 (indexing from 0, June), m = 6 and the IUCR was "0860" (retail theft). This would then return the total number of times retail theft occurred in each month of June over the past 6 years (from 2017 to 2022). Determining which month a timestamp occurs in constitutes computation on the time axis, and summing number of times that crime occurred for the given month is computation on the y-axis; thus, this is a complex trend query. The utility of this trend is that it is season-agnostic; because samples are taken from each month of the year, they are always in the same season. This means that observed trends, such as an increase or decrease in a particular crime type over time, can be attributed to something other than seasonal impacts. This query could also be extended to allow for several different codes to be input, with each corresponding to a different color on the output plot - this would allow for useful comparative analysis.
